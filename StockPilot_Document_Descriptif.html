1. Présentation Générale

StockPilot est une application web SaaS (Software as a Service) multi-tenant de gestion de stock intelligente, conçue pour répondre aux besoins des PME tunisiennes. Elle permet à plusieurs entreprises indépendantes de gérer leur inventaire de façon sécurisée et isolée sur une infrastructure partagée, avec l'assistance de l'intelligence artificielle.

4. Architecture Multi-Tenant

La sécurité multi-tenant repose sur trois couches d'isolation imbriquées, garantissant qu'aucune donnée d'une organisation ne peut être accédée par une autre, même en cas d'erreur applicative.

Requête HTTP entrante │ ▼ ┌──────────────────────────────────────────────────────┐ │ COUCHE 1 — Middleware AuthenticateTenant │ │ • Valide le token JWT │ │ • Extrait organisation_id depuis le payload │ │ • Injecte dans le conteneur Laravel (app container) │ └──────────────────────────────────────────────────────┘ │ ▼ ┌──────────────────────────────────────────────────────┐ │ COUCHE 2 — TenantScope (Eloquent Global Scope) │ │ • Applique WHERE organisation_id = ? automatiquement │ │ • Sur TOUS les modèles qui étendent BaseModel │ │ • Incontournable, même si le développeur l'oublie │ └──────────────────────────────────────────────────────┘ │ ▼ ┌──────────────────────────────────────────────────────┐ │ COUCHE 3 — Contrainte Base de Données │ │ • organisation_id NOT NULL sur toutes les tables │ │ • FK → organisations(id) │ │ • Dernier rempart au niveau SGBD │ └──────────────────────────────────────────────────────┘

Trigger Oracle — Mise à jour automatique du stock

Un trigger PL/SQL (trg_update_stock) s'exécute après chaque insertion dans stock_movements et met à jour la quantité du produit au niveau base de données, garantissant la cohérence même hors de l'application.

6. Fonctionnalités Détaillées
🔐 Authentification & Sécurité
  • Login JWT HS256 (24h + refresh 14j)
  • Protection brute-force avec verrouillage
  • 4 rôles : super_admin, admin, gestionnaire, opérateur
  • Révocation de token à la déconnexion
📦 Catalogue Produits
  • CRUD avec catégories et types dynamiques
  • Attributs personnalisables par type (text, number, date, boolean, select)
  • Calcul automatique prix TTC
  • Statuts : En stock / Alerte / Rupture

🔄 Mouvements de Stock
  • 3 types : Entrée / Sortie / Ajustement
  • Validation : sortie refusée si stock insuffisant

11. Déploiement

Développement local (sans Docker)

# Backend (PHP 8.3 requis)
cd backend
php artisan migrate
php artisan db:seed
php -d max_execution_time=0 artisan serve --port=8000

# Frontend (Node.js requis)
cd frontend
npm install
npm run dev
# → http://localhost:5173

Production (Docker Compose)

docker-compose up -d          # Démarre tous les services
php artisan migrate           # Crée les tables Oracle
php artisan db:seed           # Données initiales
# → http://localhost
Note : En production, Oracle XE 21c est requis pour les triggers PL/SQL (trg_update_stock) et les colonnes virtuelles (prix_achat_ttc, prix_vente_ttc). En développement local, SQLite est utilisé avec une logique équivalente implémentée en PHP.
